# AI 作品案例写作模板

> 用途：新增公开作品页面前，先在此模板中整理信息。只发布能够公开、已经脱敏的内容。

## 1. 项目标题

一句话标题应说明项目做了什么，例如：`面向 XX 场景的 AI 知识库检索助手`。

## 2. 项目摘要

用 60-100 字回答：

- 这是为谁解决什么问题？
- 你采取了什么方式？
- 最后形成了什么成果？

## 3. 基本信息

| 项目项 | 填写内容 |
| --- | --- |
| 项目类别 | AI 应用 / 数据分析 / 自动化 / 内容创作 / 其他 |
| 我的角色 | 例如：需求分析、方案设计、开发、测试、运营 |
| 使用工具 | 只写可公开的工具或技术名称 |
| 项目时间 | 年月或阶段，可选 |
| 展示状态 | 概念验证 / MVP / 已上线 / 已迭代 |

## 4. 业务问题

说明问题产生的背景与痛点，不写公司、客户、个人、合同或无法公开的数据。

推荐句式：

> 在 ___ 场景中，原流程存在 ___ 问题，导致 ___。因此需要一种能够 ___ 的方案。

## 5. 我的方案

将你的动作写为 3-5 步：

1. 我如何理解或拆解问题。
2. 如何准备数据、资料或知识内容。
3. 如何使用 AI 或分析工具。
4. 如何加入复核、风险控制或反馈机制。
5. 如何交付或沉淀成果。

流程概括：

```text
输入 -> 处理 -> AI/分析 -> 人工验证 -> 结果输出
```

## 6. 成果与价值

优先写可以被验证的结果：

- 交付了什么：页面、流程、报告、知识库、分析结论、演示工具等。
- 提升了什么：效率、质量、一致性、可追溯性、学习效果等。
- 指标是否来自演示数据：如为模拟样本，请明确注明。

## 7. 展示材料

发布前准备：

- 一张封面图或流程示意图。
- 1-3 张脱敏后的页面/结果截图。
- 可选：公开演示链接或公开仓库链接。

## 8. 复盘

用一段话回答：

- 哪个判断最关键？
- 如果再做一次，你会改进什么？
- 该方法还能迁移到什么场景？

## 9. 发布前检查

- [ ] 已去除公司、客户和个人信息。
- [ ] 已去除原始业务数据、账号、链接、密钥和内部路径。
- [ ] 涉及效果数字时，已标注是真实公开结果还是模拟演示结果。
- [ ] 使用的图片、字体、素材具有发布权限。
- [ ] 获得必要的公司或项目授权后再发布工作成果。
