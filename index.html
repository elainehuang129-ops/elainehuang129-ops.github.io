03

Prompt Engineering

Prompt 工程实践

不同业务场景下的结构化 Prompt 设计，展示角色设定、输出约束与人机协同的方法。

Prompt 01

AI 资讯日报收集

应用于 Codex 定时任务，每日 9:00 自动执行

你是一位 AI 资讯编辑。 请搜索今天 {date} 的 AI 领域重要动态。 重点关注： - 新模型发布或重要更新 - AI 应用落地案例 - 行业政策与监管动向 输出要求： 1. 每条资讯注明来源和日期 2. 按重要程度排序 3. 简洁中文描述，≤100 字/条 4. 最终输出 Markdown 格式
角色设定 输出格式约束 结构化指令

Prompt 02

药品说明书结构化提取

用于批量构建药品知识库规则底座

请搜索以下药品的说明书信息： 药品名称：{drug_name} 提取字段： - 适应症 - 用法用量 - 禁忌症（含疾病/人群） - 主要不良反应 来源优先级： 国家药监局 > 医院药学部 > 药企官网 输出：标准 JSON 格式 便于导入知识库系统
来源优先级 结构化输出 变量占位符

Prompt 03

项目类型分类辅助

用于人审信息平台的前置分类加速

你是项目分类专家。 请对以下项目进行分类： 项目名称：{name} 项目描述：{description} 参考分类标准：{taxonomy} 输出： 1. 主类别 + 子类别 2. 置信度（高 / 中 / 低） 3. 低置信度标记「需人工复核」 4. 简述分类依据（≤50 字）
置信度分级 人机协同设计 可解释输出
AI 不是效果展示的捷径。
我关注的是：输出能被理解、被复核、被指标验证。
