药品知识库构建

通过 Codex 联网搜索批量收集药品说明书关键字段，结构化提取后导入知识库系统，助力构建药品规则底座。

业务背景

药品知识库建设需要对大量药品说明书进行逐一整理，传统方式依赖人工录入，效率低、周期长，且字段格式难以统一。

我的方案

我设计了一套批量处理流程：通过结构化提示词检索指定药品的中国批准适应症，优先记录官方说明书或可审计官方来源，并要求固定输出适应症原文、来源、核验日期与是否可直接推断疾病的结论。

标准化药品名官方来源检索适应症原文提取唯一性判断人工复核入库

验证流程：从适应症到可用规则

药品名称不能天然等同于单一疾病。公开展示版采用严格规则：仅当中国批准适应症的官方证据明确、方向唯一且词项足够精确时，才允许进入可直接推断的规则库。

01 / CAPTURE

收集证据

保存通用名、原始适应症文本、官方来源标题/链接与核验日期。

02 / AUDIT

判断唯一性

多适应症、预防/支持/诊断性用途或证据不足的药品不直接映射疾病。

03 / REVIEW

审核入库

通过项进入规则库；存疑项保留证据并送入医学人工复核队列。

证据搜集路径 / 可审计输入

同一候选药品优先采用可复核的中国批准适应症来源

验证口径
01 官方说明书

优先保留说明书或官方医学信息页中的适应症文本。

02 官方监管来源

使用监管发布页或公开附件补充批准与变更信息。

03 结论留痕

记录来源标题、链接、核验日期及直推/送审结论。

入库必要字段

规则词项、词项类型、完整适应症、推断疾病、置信结论。

审计必要字段

判定原因、官方证据链接、审核日期、版本与复核状态。

成果

沉淀了一套“采集字段 + 官方证据 + 唯一性判断 + 人工审核”的数据加工口径，能够将批量搜集从文本搬运升级为可审计的规则准备流程。

当前公开页面展示的是验证设计与脱敏样例；授权后的原始说明书证据与正式入库记录应保留在受控环境中。

价值与沉淀

在知识库建设场景中，提示词的来源优先级和“拒绝入库”的规则同样重要：权威来源优先能降低错误，严格排除多适应症和证据不足条目则能降低误推断风险。

还可以延伸到什么

药品规则审计库

形成带证据链接、审核日期和版本记录的可部署规则清单。

适应症变更预警

定期检查新增或扩展适应症，将失效的直推规则自动送审。

审核 API 服务

向人审平台提供“可推断/需复核/禁止推断”的标准化判断结果。
